To regenerate:
```bash
python3 tests/generate_eval_corpus.py          # create if missing
python3 tests/generate_eval_corpus.py --force   # re-download + overwrite
```

## 2. OCR Accuracy Metrics

| Metric | Definition | Target (clean) |
|---|---|---|
| **Word Error Rate (WER)** | Levenshtein distance at word level / total words | < 5% |
| **Character Error Rate (CER)** | Levenshtein distance at character level / total chars | < 2% |
| **Word Recall** | Ground-truth words appearing in OCR output | > 95% |
| **Word Precision** | OCR output words appearing in ground truth | > 90% |
| **Mean Confidence** | Average Tesseract per-word confidence | > 80 |

Implemented in `src/test/java/com/trulyfreeocr/eval/Levenshtein.java` (static helpers)
and `OcrAccuracyTest.java` (per-page + aggregate report).

**Aggregate (10 pages):**
- Aggregate WER: 15.1%
- Aggregate CER: 11.1%
- Word recall: 98.7%
- Word precision: 98.7%
- Mean confidence: 92.8
- Total time: ~848s (~85s/page)
- Pages with WER > 5%: 4 / 10 (title, etymology, extracts — non-standard layouts)

### Analysis

- **Regular prose pages** (pages 5–10) achieve **WER 0.9%–3.2%**, well within the <5% target. CER is <1.2%.
- **Title, etymology, and extracts pages** (pages 1–4, ~40% of this corpus) inflate the aggregate due to multi-column layouts, mixed languages, punctuation-heavy extracts, and unusual typography. These pages are not representative of typical document content.
- **Recall and precision** are excellent across all pages (≥96%), indicating Tesseract correctly identifies almost all words and introduces very few false positives.
- **Mean confidence** is high and stable (89.5–94.7).
- **Per-page time** varies 44–136s at 300 DPI, averaging ~85s. Expected to scale linearly with page count.

## 4. Segmentation Quality

| Metric | How |
|---|---|
| **Blank page false positives** | Foreground pixel % on `blank.pdf` (already tested: < 5%) |
| **Text page recall** | Are all rendered text pixels captured in the mask? |
| **Inpaint quality** | % of foreground pixels that differ between original and cleaned background (already tested: > 50%) |

## 5. End-to-End Pipeline

| Test | Verification |
|---|---|
| **Searchability** | Extract text from output PDF, compare to ground truth per page |
| **Compression ratio** | Output / input file size, with and without jbig2enc |
| **Metadata round-trip** | Bookmark count, annotation count, document info fields |
| **CCITT G4 fallback** | Output size and quality when jbig2enc is unavailable |
