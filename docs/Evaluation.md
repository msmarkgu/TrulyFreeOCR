## 1. Ground Truth Corpus

Generated by `tests/generate_eval_corpus.py` from public-domain text (Project Gutenberg — The Adventures of Sherlock Holmes).

| Corpus PDF | Pages | GT Words | Notes |
|---|---|---|---|
| `sherlock-holmes-010p.pdf` | 10 | 4,291 | Quick smoke-test |

| `sherlock-holmes-050p.pdf` | 50 | 23,487 | Short document |
| `sherlock-holmes-100p.pdf` | 100 | 48,121 | Typical book chapter |

The generator renders clean image-only PDFs (no selectable text) with a serif font
at 300 DPI. Per-page word lists are saved alongside each PDF as JSON ground truth.

To regenerate:
```bash
python3 tests/generate_eval_corpus.py --corpus sherlock-holmes          # create if missing
python3 tests/generate_eval_corpus.py --corpus sherlock-holmes --force   # re-download + overwrite
```

## 2. OCR Accuracy Metrics

| Metric | Definition | Target (clean) |
|---|---|---|
| **Word Error Rate (WER)** | Levenshtein distance at word level / total words | < 5% |
| **Character Error Rate (CER)** | Levenshtein distance at character level / total chars | < 2% |
| **Word Recall** | Ground-truth words appearing in OCR output | > 95% |
| **Word Precision** | OCR output words appearing in ground truth | > 90% |
| **Mean Confidence** | Average Tesseract per-word confidence | > 80 |

Implemented in `src/test/java/com/trulyfreeocr/eval/Levenshtein.java` (static helpers)
and `OcrAccuracyTest.java` (per-page + aggregate report).

**Aggregate (10 pages):**
- Aggregate WER: 5.4%
- Aggregate CER: 1.6%
- Word recall: 98.8%
- Word precision: 97.7%
- Mean confidence: 91.6
- Total time: ~119s (~12s/page)
- Pages with WER > 5%: 5 / 10

### Analysis

- **All pages are standard prose** (no multi-column layouts, no mixed languages), yet WER hovers around 4–7%, compared to 0.9–3.2% on the old Moby Dick corpus. The difference likely reflects the more compact line spacing and smaller font in the Sherlock Holmes rendering.
- **Word recall and precision** remain strong across all pages (≥94% recall, ≥94% precision), indicating Tesseract correctly identifies almost all words with few false positives.
- **Mean confidence** is high and stable (90.0–93.6).
- **Per-page time** is much faster than the old corpus (~12s vs ~85s) — likely due to the smaller rendered page area from the different PDF layout.

## 4. Segmentation Quality

| Metric | How |
|---|---|
| **Blank page false positives** | Foreground pixel % on `blank.pdf` (already tested: < 5%) |
| **Text page recall** | Are all rendered text pixels captured in the mask? |
| **Inpaint quality** | % of foreground pixels that differ between original and cleaned background (already tested: > 50%) |

## 5. End-to-End Pipeline

| Test | Verification |
|---|---|
| **Searchability** | Extract text from output PDF, compare to ground truth per page |
| **Compression ratio** | Output / input file size, with and without jbig2enc |
| **Metadata round-trip** | Bookmark count, annotation count, document info fields |
| **CCITT G4 fallback** | Output size and quality when jbig2enc is unavailable |
